Wait...why are we calling it index.html?

  • index.html is the default filename a web browser looks for when pointed to any directory.
  • You won't necessarily see it reflected in the URL — we just see http://girldevelopit.com, but the browser is displaying http://girldevelopit.com/index.html. Sneaky!
  • You should have an index.html file for every directory on your site. If not, users will see a list of all your filenames, like this:
You can name your CSS stylesheets whatever you want, but styles.css is the standard.

Inline vs Block

So far, we have mostly seen "block" elements

They appear on the next line, like paragraphs

  • Block elements: p, h1, ul, li, almost everything else


There are also "inline" elements

They appear on the same line that they are written on.

  • Inline elements: img, a, br, em, strong

Inline vs. Block

We can tell an element to display inline or block with CSS:


              nav ul li {
              display: inline;
          	}

Box Model

The CSS box model is essentially a box that wraps around HTML elements, and it consists of: margins, borders, padding, and the actual content.

The box model allows us to place a border around elements and space elements in relation to other elements.

Definition from w3schools

Alternate Box Model

Padding

Space between the border and the content

Adds to the total width of the box.

Padding (all sides)

15 pixels on all sides 

              padding: 15px;

Padding Top

10 pixels on top only 

              padding-top: 10px;

One Value


          .all_sides{
            padding: 15px;

Two Values


            .topANDbottom_rightANDleft {

Multiple Values


            .top_right_bottom_left {

IDs vs. Classes

ID -- Should only apply to one element on a webpage (like #top -- there is only one top of the page). Can be used to link to different parts of a single page. (Example: npr.org).


      <p id = "this_ID">
          

      #this_ID { color:red; }

Class -- Lots of elements can have the same class. I.E. There can be many warnings on one webpage.


    <p class="this_class">
    <a class="this_class">
          

    .this_class { color:#333; }

IDs vs. Classes

Another way to think about it:

Like a department store, you just have one women's department, one men's department, one kids' department. These departments are like ID's.


Inside each department are all kinds of products: sweaters, shirts, socks, in all colors and sizes. There are multiples of these, so these are like classes.

    Pseudo-classes, more CSS for links

    Changing the format of a link when you hover over it is accomplished by using pseudo-classes.


    CSS pseudo-classes are used to add special effects to some selectors.

    Syntax: 
    
            selector:pseudo-class
            {
              property:value; 
            }
    Example: 
    
            a:link
            {
              text-decoration: none;
            }

    Pseudo-classes, more CSS for links

    
            a:link  {color:#FF0000;} /* unvisited link */
            a:visited {color:#00FF00;} /* visited link */
            a:focus {color: #FF00FF;} /* link currently in focus */
            a:hover {color:#FF00FF;} /* mouse over link */
            a:active  {color:#0000FF;} /* selected link */
    * a:hover MUST come after a:link and a:visited in the CSS definition in order to be effective! * a:active MUST come after a:hover in the CSS definition in order to be effective!

    • If you buy a web hosting plan, you can upload your files to a webserver.

    • A webserver is a computer that is accessible online

    • The webserver stores your files

    How does someone find my site?

    • A domain name is an address you buy

    • The address points anywhere you want, in this case, the files on the server.

    Server + Domain + You
